# PDFShrink 发布前手测清单

每次 release 前，在 macOS arm64 / x64 与 Windows 11 上各跑一遍。

## 基础流程
- [ ] 拖入 fixture 扫描件 → 默认档位 → 输出体积 < 输入，可在系统 PDF 阅读器打开
- [ ] 拖入纯文本 PDF → 输出 < 输入
- [ ] 输出文件名为 `{原名}_compressed.pdf`，与原文件同目录
- [ ] 输出已存在时，新文件名带时间戳后缀

## 档位切换
- [ ] 高质量 → 输出比标准大但比原文件小
- [ ] 极致压缩 → 输出最小
- [ ] 自定义 dpi=180 → 与档位 200 接近

## 错误路径
- [ ] 拖入 .docx → 红色提示 "请拖入 PDF 文件"
- [ ] 拖入加密 PDF → 红色提示包含 "压缩失败"
- [ ] 自定义 dpi 输入 9999 → 输入框校验提示
- [ ] 自定义 dpi 输入字母 → 输入框拒绝

## 多文件
- [ ] 同时拖入 3 个 PDF → 提示 "已忽略其余 2 个"，第一个被压缩

## 网络
- [ ] 关闭 wifi → 拖入 PDF → 仍正常工作（更新检查静默失败）

## 输出验证
- [ ] 用 `qpdf --check <output.pdf>` 验证 → No syntax errors
